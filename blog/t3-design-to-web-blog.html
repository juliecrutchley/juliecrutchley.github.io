What is a responsive site and why is responsiveness important?

Responsiveness refers to a website that will 'respond' to the screen size it is being viewed on, i.e. if a mobile screen is being used, it will respond by altering the layout to best fit that screen. For a mobile screen, it might minimize image and text sizing to fit better and collapse large menus to allow you to easily view and navigate the site.
Responsiveness is vital for websites, now that it is common for users to access the web via multiple different screen sizes every day, i.e. on larger screens at work, on laptop screens at home, and on mobile screens throughout the day.

What is mobile first design and why is this important?

Mobile first design refers to starting the design process of a website with the mobile display design first. As the mobile screen is the smallest, the design will be the most basic, showcasing the most important elements of the site. Think of it as an elevator pitch for a business - it has to be short, sharp and have all the important stuff in a small space.
It is important to start with this first and then branch out to the design of the larger screen sizes because then you can expand on elements, menus, add more images etc. If you try to design the website from largest to smallest, you will end up trying to cram too much content into the smaller area.

What are frameworks and what are their pros and cons?

Frameworks are awesome! They are pre-formatted files with basic website content and styling settings already set up which can be used as a base on which to begin building a website. Think of them as a master blueprint for a basic house which an architect could re-use again and again, by then altering the number of rooms, floors, windows etc.

What is a wireframe and why do we use them?

A wireframe is a basic sketch of the layout of a site, often scribbled with pen and paper to quickly create several layout options. The layout is usually setout with 12 columns and basic boxes, in reference to how the HTML code would need to be written.
They are usually created without any text, colors or any major styling in an effort to not be distracted by those factors and just focus on the 'bare bones' of the website.

What aspects of wireframes did you find difficult to implement and why?

Aligning the images as per the wireframes posed the most difficult task, they kept wanting to overflow into other areas. Got them into line in the end!
